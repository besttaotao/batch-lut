# 批量色彩还原工具 (Batch LUT Restoration Tool)

一款专为 **DJI Osmo Action 5/4**、**Pocket 3** 等设备开发的桌面端批量色彩还原工具。支持 D-Log M 素材一键转标准色彩（Rec.709），并具备自适应原片规格的无损导出能力。

---

## 🌟 核心特性

- **规格自适应**：自动识别原片分辨率、帧率及码率，确保还原后的视频规格 1:1 对齐。
- **全平台硬件加速**：支持 NVIDIA (NVENC) 和 AMD (AMF) 显卡加速，以及通用 CPU 模式。
- **无损画质**：默认采用 H.265 (HEVC) 编码，支持 100Mbps+ 比特率匹配，几乎无损还原。
- **零环境依赖**：打包后内置 FFmpeg，用户无需安装任何环境。

---

## 🚀 操作手册

1.  **添加视频**：拖拽 MP4 或点击中心区域选择文件。
2.  **选择 LUT**：选取对应的 `.cube` 还原文件。
3.  **设置目录**：指定导出位置（支持防覆盖重命名）。
4.  **选择模式**：根据显卡型号选择对应的加速模式。
5.  **一键导出**：点击开始，支持随时中止并自动清理缓存。

## 🛠️ 开发者指南 (Developer Guide)

如果你希望在本地克隆并运行本项目，请务必阅读以下配置说明。

### 1. 环境准备
- **Node.js**: 建议 v18.0 或更高版本 (推荐 v22+)。
- **Git**: 用于版本管理。

### 2. 核心依赖：FFmpeg 二进制文件
本项目由于法律及体积原因，源码仓库通常不包含 FFmpeg 执行文件。**在运行或打包前，必须手动配置：**
1.  下载适用于 Windows 的 [FFmpeg Release](https://www.gyan.dev/ffmpeg/builds/)。
2.  在项目根目录下创建目录：`resources/bin/`。
3.  将下载包中的 `ffmpeg.exe` 和 `ffprobe.exe` 放入该目录下。
    - *路径结构示例：* `batch-lut-app/resources/bin/ffmpeg.exe`

### 3. 项目安装与运行
```bash
# 1. 安装依赖
npm install

# 2. 启动开发模式预览
npm run dev

# 3. 打包生成正式版安装包 (.exe)
npm run build:win
```

### 4. 项目结构说明
- `src/main/index.ts`: **主进程**。负责调用 FFmpeg、处理文件系统交互及显卡加速逻辑。
- `src/renderer/src/`: **渲染进程 (Vue)**。负责 UI 展现、拖拽交互及进度展示。
- `src/preload/index.ts`: **预加载脚本**。作为主进程与渲染进程的通信桥梁。
- `resources/`: 存放图标及内置的二进制引擎。
